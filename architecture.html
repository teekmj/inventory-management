End-to-End Data Flow
User changes filter
FilterBar.vue dropdown
›
useFilters() ref updated
selectedPeriod / selectedLocation / selectedCategory / selectedStatus
›
watch() fires in view
watch([...filters], loadData)
›
getCurrentFilters()
maps refs → { warehouse, category, month, status }
›
api.js builds request
URLSearchParams · skips "all" values
›
FastAPI receives params
GET /api/orders?warehouse=Tokyo&month=Q1-2025
›
apply_filters() runs
warehouse · category · status exact match
›
filter_by_month() runs
QUARTER_MAP expands Q1-2025 → ['2025-01','2025-02','2025-03']
›
Pydantic validates
typed response model serialized to JSON
›
Component renders
computed properties derive display data
API Endpoints
Method Path warehouse category status month Notes
GET /api/inventory yes yes no no No time dimension on inventory
GET /api/inventory/{id} — Single item detail
GET /api/orders yes yes yes yes Full filtering · quarter expansion
GET /api/orders/{id} — Single order detail
GET /api/dashboard/summary yes yes yes yes Aggregated KPIs for Dashboard
GET /api/demand no filters Demand forecasts
GET /api/backlog no filters Injects has_purchase_order flag
GET /api/spending/summary no filters Total spend by category
GET /api/spending/monthly no filters Monthly cost breakdown
GET /api/spending/categories no filters
GET /api/spending/transactions no filters
GET /api/reports/quarterly no filters Groups orders by quarter
GET /api/reports/monthly-trends no filters Monthly revenue & delivery metrics
GET /api/tasks no filters User task list
POST /api/tasks — Create task
PATCH /api/tasks/{id} — Update task status
DELETE /api/tasks/{id} — Delete task
POST /api/purchase-orders — Create PO from backlog item
GET /api/purchase-orders/{backlogItemId} — Get PO for a backlog item
